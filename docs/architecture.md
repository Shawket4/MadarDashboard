# Sufrix Dashboard — Architecture (rebuild v2)

The frontend was rebuilt from scratch on a modern stack while **preserving the data
contract**. This doc explains the structure and conventions.

## Stack
- **React 19 + Vite 6 + Tauri 2 + TypeScript**
- **Tailwind CSS v4** (CSS-first `@theme`, Oxide engine) + **canonical shadcn/ui**
- **TanStack Router** (file-based, typed search params) — replaces React Router
- **TanStack Query** + **Orval** (generated hooks) + **Axios** — the data layer
- **Zustand** (auth / app / scope stores)
- **Motion** (animation), **react-hook-form + zod** (forms), **i18next** (en/ar, RTL)
- **Recharts**, **lucide-react**, **sonner**, **cmdk**, **exceljs**

## Folder layout


## Data layer (do not rewrite without reason)
- **Orval** generates hooks + zod from `../SufrixRust/openapi.json` into `src/data/api/generated`.
  Regenerate with `npm run generate:api`. Consume generated hooks **directly**
  (e.g. `useLogin`, `useListBranches`) and invalidate with the generated
  `getXxxQueryKey()` helpers — no manual key map.
- **Axios** (`data/api/client.ts`) injects `Authorization` + `X-Org-Id` + `X-Branch-Id`
  from an ambient context the stores write to. A 401 triggers sign-out.
- **Stores**: `auth` (token/user, persisted `sufrix.auth`), `app` (org/branch/lang/sidebar,
  `sufrix.app`), `scope` (branch + Cairo date range, `sufrix-scope`). They self-wire on import
  (see `main.tsx` side-effect imports).

## Routing
- File-based. Auth is enforced in `routes/_app/route.tsx` `beforeLoad` → `requireAuth`
  (`lib/auth-guard.ts`). `login.tsx` bounces authed users to `/`.
- Scope (branch / preset / from / to) is declared as **validated search params** on `/_app`.
- `ComingSoon` placeholders fill routes that are rebuilt in later phases.

## Adding a page (later phases)
1. Replace the route's `makeComingSoon(...)` with `component: MyPage`.
2. Build `features/<domain>/my-page.tsx` using `Page` + `PageHeader` and shadcn/composite components.
3. Use generated Orval hooks for data; `fmt*` helpers from `lib/format`; `t()` for all strings.
4. Keep it responsive (mobile-first) and RTL-safe (logical properties — `ps/pe/ms/me/start/end`).
